---
title: 'Create Consultation'
api: 'POST /consultations/create'
description: 'Schedule a new consultation and automatically create a pending payment record'
---

Creates a new consultation in the system. This endpoint validates that both the patient (by CPF) and doctor (by CRM) exist in the database before scheduling. Upon successful creation, a payment record is automatically created with PENDING status, and a confirmation email is sent.

## Request Body

<ParamField body="patient" type="object" required>
  Patient information object
  
  <Expandable title="Patient properties">
    <ParamField body="cpf" type="string" required>
      Patient's CPF (11 digits) - must exist in the database
    </ParamField>
  </Expandable>
</ParamField>

<ParamField body="doctor" type="object" required>
  Doctor information object
  
  <Expandable title="Doctor properties">
    <ParamField body="crm" type="string" required>
      Doctor's CRM number (up to 9 characters) - must exist in the database
    </ParamField>
  </Expandable>
</ParamField>

<ParamField body="dateTime" type="string" required>
  Consultation date and time in ISO 8601 format (e.g., "2024-03-15T14:30:00")
</ParamField>

<ParamField body="isUrgent" type="boolean" required>
  Whether this is an urgent consultation
</ParamField>

<ParamField body="observation" type="string">
  Additional notes or observations for the consultation
</ParamField>

## Side Effects

### Payment Creation
A payment record is automatically created with:
- Amount set to the doctor's `consultationValue`
- Status set to `PENDING`
- Linked to the created consultation

### Email Notification
A confirmation email is sent to `delivered@resend.dev` with:
- Subject: "Consulta confirmada"
- Body: "Sua consulta foi marcada para [dateTime]"
