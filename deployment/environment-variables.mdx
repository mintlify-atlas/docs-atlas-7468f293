## Overview

Med Agenda uses environment variables for configuration, following the [12-factor app](https://12factor.net/) methodology. This allows the same application build to be deployed across different environments.

## Configuration Sources

Environment variables can be provided through:

- **System environment variables**: `export VAR=value`
- **`.env` file**: For local development (Docker Compose)
- **`application.properties`**: Default values and variable references
- **Docker**: `-e` flag or `env_file` in docker-compose.yml
- **Cloud platforms**: Platform-specific environment variable settings

## Required Variables

<Warning>
These variables MUST be set for the application to function properly.
</Warning>

### Database Configuration

<ParamField path="SPRING_DATASOURCE_URL" type="string" required>
  PostgreSQL JDBC connection string
  
  **Format**: `jdbc:postgresql://host:port/database?parameters`
  
  **Example**:
  ```bash
  jdbc:postgresql://localhost:5432/medagenda?sslmode=disable
  ```
  
  **Production Example (Neon.tech)**:
  ```bash
  jdbc:postgresql://user:pass@ep-xxx.region.aws.neon.tech/medagenda?sslmode=require
  ```
</ParamField>

<ParamField path="SPRING_DATASOURCE_USERNAME" type="string" required>
  Database username
  
  **Example**:
  ```bash
  postgres
  ```
</ParamField>

<ParamField path="SPRING_DATASOURCE_PASSWORD" type="string" required>
  Database password
  
  **Example**:
  ```bash
  your_secure_password
  ```
  
  <Warning>
  Never commit this to version control. Use secrets management in production.
  </Warning>
</ParamField>

### Payment Integration

<ParamField path="ABACATE-KEY" type="string" required>
  Abacate Pay API key for payment processing
  
  **Example**:
  ```bash
  abct_live_xxxxxxxxxxxx
  ```
  
  <Warning>
  Keep this secret secure. Do not expose in logs or client-side code.
  </Warning>
</ParamField>

## Optional Variables

### Server Configuration

<ParamField path="SERVER_PORT" type="integer" default="8080">
  Port number for the application server
  
  **Example**:
  ```bash
  SERVER_PORT=8080
  ```
  
  <Note>
  If running in Docker, ensure the container exposes this port.
  </Note>
</ParamField>

### Database Pool Settings

<ParamField path="DB_POOL_MAX" type="integer" default="10">
  Maximum number of connections in the HikariCP connection pool
  
  **Recommended values**:
  - Development: `5-10`
  - Production: `20-50`
  - High traffic: `50-100`
  
  **Example**:
  ```bash
  DB_POOL_MAX=20
  ```
</ParamField>

### JPA/Hibernate Settings

<ParamField path="JPA_DDL_AUTO" type="string" default="update">
  Hibernate DDL auto mode for schema management
  
  **Options**:
  - `update`: Auto-update schema (development)
  - `validate`: Validate schema only (production)
  - `create`: Drop and recreate schema (testing only)
  - `none`: No schema management
  
  **Example**:
  ```bash
  JPA_DDL_AUTO=validate
  ```
  
  <Warning>
  Use `validate` or `none` in production to prevent unintended schema changes.
  </Warning>
</ParamField>

<ParamField path="JPA_SHOW_SQL" type="boolean" default="false">
  Enable SQL query logging
  
  **Example**:
  ```bash
  JPA_SHOW_SQL=true
  ```
  
  <Note>
  Set to `true` for debugging, but disable in production for performance.
  </Note>
</ParamField>

### JVM Options (Docker only)

<ParamField path="JAVA_OPTS" type="string" default="">
  Custom JVM options for memory, garbage collection, etc.
  
  **Example**:
  ```bash
  JAVA_OPTS="-Xms512m -Xmx2g -XX:+UseG1GC -XX:MaxGCPauseMillis=200"
  ```
  
  **Common options**:
  - `-Xms`: Initial heap size
  - `-Xmx`: Maximum heap size
  - `-XX:+UseG1GC`: Use G1 garbage collector
  - `-XX:+HeapDumpOnOutOfMemoryError`: Create heap dump on OOM
</ParamField>

## Email Service Configuration

Med Agenda uses [Resend](https://resend.com) for sending email notifications.

<Warning>
The Resend API key is currently hardcoded in the source (`EmailService.java:16`). For production, this should be configured via environment variable.
</Warning>

### Recommended Email Variables

While not currently supported, these should be added for production:

```bash
# Email service configuration (to be implemented)
RESEND_API_KEY=re_xxxxxxxxxxxx
RESEND_FROM_EMAIL=noreply@yourdomain.com
```

**Current behavior**:
- API Key: Hardcoded as `re_6gEiCUYB_D8rKKD2XZPJRhUXQCh6gQwpX`
- From Email: `onboarding@resend.dev`

<Note>
Consider refactoring `EmailService.java` to read these from environment variables for better security.
</Note>

## Configuration Files

### application.properties

The default configuration file with variable references:

```properties
spring.application.name=gestaoConsultasMedicas

# Database
spring.datasource.url=${SPRING_DATASOURCE_URL:jdbc:postgresql://localhost:5432/postgres?sslmode=disable}
spring.datasource.username=${SPRING_DATASOURCE_USERNAME:postgres}
spring.datasource.password=${SPRING_DATASOURCE_PASSWORD:postgres}
spring.datasource.hikari.maximum-pool-size=${DB_POOL_MAX:10}

# JPA/Hibernate
spring.jpa.hibernate.ddl-auto=${JPA_DDL_AUTO:update}
spring.jpa.show-sql=${JPA_SHOW_SQL:false}
spring.jpa.database-platform=org.hibernate.dialect.PostgreSQLDialect

# Server
server.port=${SERVER_PORT:8080}

# Payment
abacatepay.api-key=${ABACATE-KEY}
```

## Setting Environment Variables

### Linux/macOS

<CodeGroup>
```bash Export (Current session)
export SPRING_DATASOURCE_URL="jdbc:postgresql://localhost:5432/medagenda"
export SPRING_DATASOURCE_USERNAME="postgres"
export SPRING_DATASOURCE_PASSWORD="your_password"
export ABACATE-KEY="your_api_key"
```

```bash .bashrc/.zshrc (Persistent)
echo 'export SPRING_DATASOURCE_URL="jdbc:postgresql://localhost:5432/medagenda"' >> ~/.bashrc
source ~/.bashrc
```

```bash Inline (One-time)
SPRING_DATASOURCE_URL="..." java -jar app.jar
```
</CodeGroup>

### Windows

<CodeGroup>
```powershell PowerShell (Current session)
$env:SPRING_DATASOURCE_URL="jdbc:postgresql://localhost:5432/medagenda"
$env:SPRING_DATASOURCE_USERNAME="postgres"
$env:SPRING_DATASOURCE_PASSWORD="your_password"
```

```cmd Command Prompt (Current session)
set SPRING_DATASOURCE_URL=jdbc:postgresql://localhost:5432/medagenda
set SPRING_DATASOURCE_USERNAME=postgres
set SPRING_DATASOURCE_PASSWORD=your_password
```

```powershell System Environment Variables (Persistent)
[System.Environment]::SetEnvironmentVariable('SPRING_DATASOURCE_URL', 'jdbc:postgresql://localhost:5432/medagenda', 'User')
```
</CodeGroup>

### Docker

<CodeGroup>
```bash Docker CLI
docker run -d \
  -e SPRING_DATASOURCE_URL="jdbc:postgresql://db:5432/medagenda" \
  -e SPRING_DATASOURCE_USERNAME="postgres" \
  -e SPRING_DATASOURCE_PASSWORD="postgres" \
  -e ABACATE-KEY="your_key" \
  med-agenda:latest
```

```yaml Docker Compose
services:
  api:
    image: med-agenda:latest
    environment:
      SPRING_DATASOURCE_URL: jdbc:postgresql://db:5432/medagenda
      SPRING_DATASOURCE_USERNAME: postgres
      SPRING_DATASOURCE_PASSWORD: postgres
      ABACATE-KEY: your_key
```

```yaml Docker Compose (.env file)

  api:
    image: med-agenda:latest
    env_file:
      - .env
```
</CodeGroup>

## Security Best Practices

<Warning>
Follow these security practices to protect sensitive configuration:
</Warning>

### 1. Never Commit Secrets

Add `.env` to `.gitignore`:

```bash .gitignore
# Environment variables
.env
.env.local
.env.production
*.env

# Application properties with secrets
application-local.properties
application-prod.properties
```

### 2. Use Different Keys per Environment

| Environment | Key Type |
|-------------|----------|
| Development | Test/sandbox keys |
| Staging | Test keys |
| Production | Live production keys |

### 3. Rotate Secrets Regularly

- Change database passwords periodically
- Rotate API keys every 90 days
- Update credentials after team member changes

### 4. Use Secrets Management

For production deployments, use dedicated secrets management:

<Tabs>
  <Tab title="AWS Secrets Manager">
    ```bash
    # Retrieve secret from AWS
    export DB_PASSWORD=$(aws secretsmanager get-secret-value \
      --secret-id prod/medagenda/db-password \
      --query SecretString \
      --output text)
    ```
  </Tab>
  
  <Tab title="HashiCorp Vault">
    ```bash
    # Retrieve secret from Vault
    export DB_PASSWORD=$(vault kv get -field=password secret/medagenda/db)
    ```
  </Tab>
  
  <Tab title="Docker Secrets">
    ```yaml docker-compose.yml
    services:
      api:
        image: med-agenda:latest
        secrets:
          - db_password
    
    secrets:
      db_password:
        external: true
    ```
  </Tab>
  
  <Tab title="Kubernetes Secrets">
    ```yaml
    apiVersion: v1
    kind: Secret
    metadata:
      name: medagenda-secrets
    type: Opaque
    data:
      db-password: <base64-encoded-password>
    ```
  </Tab>
</Tabs>

### 5. Restrict File Permissions

```bash
# Make .env readable only by owner
chmod 600 .env

# Verify permissions
ls -la .env
# Should show: -rw------- (600)
```

### 6. Validate Configuration

Create a startup check to ensure required variables are set:

```java
@Component
public class ConfigValidator implements ApplicationListener<ApplicationReadyEvent> {
    @Value("${ABACATE-KEY:}")
    private String apiKey;
    
    @Override
    public void onApplicationEvent(ApplicationReadyEvent event) {
        if (apiKey == null || apiKey.isEmpty()) {
            throw new IllegalStateException("ABACATE-KEY environment variable not set");
        }
    }
}
```

## Verification

Verify your configuration is loaded correctly:

### Check Environment Variables

<CodeGroup>
```bash Linux/macOS
# View all Java environment variables
env | grep -E '(SPRING|SERVER|JPA|DB_|JAVA_)'

# Test database connection
pg_isready -h your_host -p 5432 -U your_user
```

```bash Docker
# View container environment variables
docker exec med-agenda env | grep -E '(SPRING|SERVER|JPA)'

# Check running configuration
docker exec med-agenda java -XshowSettings:properties -version
```
</CodeGroup>

### Application Startup Logs

Check logs for configuration values:

```bash
# Look for connection pool settings
docker-compose logs api | grep -i hikari

# Verify JPA DDL mode
docker-compose logs api | grep -i "ddl-auto"

# Check server port
docker-compose logs api | grep -i "Tomcat started on port"
```

## Troubleshooting

<AccordionGroup>
  <Accordion title="Variable not being read">
    Ensure variable is exported before running:
    ```bash
    export VAR=value
    echo $VAR  # Verify it's set
    ```
    
    For Docker, ensure it's in `.env` file or passed with `-e` flag.
  </Accordion>
  
  <Accordion title="Database connection fails">
    Verify connection string format:
    ```bash
    # Should be JDBC format, not PostgreSQL native
    # Correct:
    jdbc:postgresql://host:5432/db
    
    # Incorrect:
    postgresql://host:5432/db
    ```
  </Accordion>
  
  <Accordion title="Port already in use">
    Change the server port:
    ```bash
    export SERVER_PORT=8081
    ```
    
    Or kill process using the port:
    ```bash
    lsof -ti:8080 | xargs kill -9
    ```
  </Accordion>
  
  <Accordion title="Special characters in password">
    URL-encode special characters in connection string or use separate username/password variables:
    ```bash
    # Preferred: separate variables
    SPRING_DATASOURCE_USERNAME=user
    SPRING_DATASOURCE_PASSWORD="p@ssw0rd!"
    
    # Alternative: URL encode in connection string
    # @ = %40, ! = %21, etc.
    ```
  </Accordion>
</AccordionGroup>
